# 可复制经验提取方法论

> 本文档定义了如何从分析结果中提取可操作、可复制的增长经验。

## 1. 经验提取原则

### 1.1) 可复制性判定标准

一条经验是否"可复制"，需要满足：

| 标准 | 说明 |
|------|------|
| ✅ 可操作 | 有具体的步骤和参数（不是"发好内容"这种空话） |
| ✅ 可量化 | 有数据支撑（不是"感觉有效"） |
| ✅ 可适应 | 不依赖特殊资源（如"有 100 万粉丝的朋友转发"） |
| ✅ 可持续 | 不是短期 hack（如"抽奖送 iPhone"） |

### 1.2) 经验分级

| 级别 | 定义 | 复制难度 |
|------|------|---------|
| 🟢 **直接复制** | 有固定公式，换内容即可 | 低 |
| 🟡 **需要适配** | 有框架但需根据自身情况调整 | 中 |
| 🔴 **仅供参考** | 依赖特殊资源或时机，难以复制 | 高 |

## 2. 经验提取维度

### 2.1) 内容公式提取

```
分析高互动帖的共同结构：

标题模式：
  - "X 个你不知道的 Y 技巧"
  - "我花了 Z 小时才搞明白的 W"
  - "说个得罪人的话：..."
  - "Day N：..."

正文结构：
  - 钩子句 → 核心观点 → 展开说明 → CTA
  - 问题 → 痛点 → 解决方案 → 效果
  - 反直觉观点 → 论据 → 转折 → 结论

CTA 模式：
  - 提问式："你怎么看？"
  - 资源式："需要的扣 1"
  - 互动式："你遇到过类似的情况吗？"
```

### 2.2) 发帖节奏提取

```
提取参数：
  - 日均发帖量：X 条
  - 短推 : Thread 比例：X : Y
  - 原创 : 互动（回复/引用）比例：X : Y
  - 最佳发帖时间窗口：HH:MM - HH:MM（时区）
  - 发帖间隔：最少 X 小时

输出模板：
  "每天发 X 条，其中 Y 条是原创观点，Z 条是互动回复。
   Thread 频率：每周 X 条。
   最佳发帖时间：XX:00-XX:00（你的目标时区）。"
```

### 2.3) 互动策略提取

```
提取参数：
  - 每日回复他人推文数：X 条
  - 回复策略：在大 V 帖下做有深度的回复
  - DM 策略：是否使用 Auto DM
  - 引用转发策略：引用时是否加评论

输出模板：
  "互动策略：每天花 X 分钟在目标大 V 的帖下做高质量回复。
   回复公式：认同 + 补充观点 + 提问。
   避免纯夸、纯抬杠、无信息量回复。"
```

### 2.4) 话题选择策略

```
提取参数：
  - 核心话题（占 70%+ 发帖量的）：[列表]
  - 偶尔涉及的边缘话题：[列表]
  - 蹭热点的方式：是否蹭 / 怎么蹭

输出模板：
  "核心话题：X（占 60%）、Y（占 25%）、Z（占 15%）。
   热点策略：只在 X 领域内的热点上发言，不跨界。
   禁忌话题：从不涉及 [列表] 以避免争议。"
```

### 2.5) 视觉风格提取

```
提取参数：
  - 是否使用统一配色/模板
  - 图片使用频率：X% 的推文带图
  - 视频使用频率：X% 的推文带视频
  - 截图风格：代码截图 / 数据截图 / 概念图

输出模板：
  "视觉策略：X% 推文配图，使用 [风格] 风格。
   常见图类型：[列表]。
   Thread 配图规律：[说明]。"
```

## 3. 30 天复制计划生成规则

### 3.1) 计划结构

```
第 1 周：基础设施
  Day 1-2：[具体动作]
  Day 3-5：[具体动作]
  Day 6-7：[具体动作 + 第一周复盘]

第 2 周：找到节奏
  Day 8-10：[具体动作]
  Day 11-12：[具体动作]
  Day 13-14：[第二周复盘 + 调整]

第 3 周：放大增长
  Day 15-18：[具体动作]
  Day 19-21：[第三周复盘]

第 4 周：固化习惯
  Day 22-28：[具体动作]
  Day 29-30：[月度复盘 + 下月计划]
```

### 3.2) 计划生成原则

1. **先易后难**：第 1 周只做最简单的动作
2. **每天有具体任务**：不说"发好内容"，说"发 1 条观点推文 + 回复 5 条大V推文"
3. **每周有复盘节点**：到周日出检查清单
4. **留调整空间**：根据实际数据反馈调整
5. **标注优先级**：必须做 ✅ vs 建议做 💡

## 4. 经验沉淀到知识库

每次分析完成后，将可复用经验按以下格式存入 `knowledge-base/patterns/`：

```markdown
# 增长策略：[策略名称]

## 来源
- 账号：@HANDLE
- 分析日期：YYYY-MM-DD
- 粉丝量级：XX,XXX

## 策略描述
[一段话描述这个策略是什么]

## 适用条件
- 行业/领域：[说明]
- 粉丝量级：[适合起步/增长/稳定期]
- 内容能力要求：[说明]

## 具体步骤
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

## 预期效果
- 互动率提升：约 X%
- 涨粉速度：约 X 个/天

## 风险提示
- [风险 1]
- [风险 2]
```
